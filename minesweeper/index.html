🔐 הרחבה 2 שיישמת – הצפנה

פרויקט שלך כולל הצפנה לנתונים רגישים (סיסמאות/נתוני משתמש). זוהי הרחבה לסעיף שירותי הרשת. 

// הצפנה לנתונים רגישים לפני שמירה / שליחה לשרת
public class EncryptionUtil {
    private static final String ALGORITHM = "AES";
    
    public static String encrypt(String data, String key) throws Exception {
        // שימוש ב-AES להצפנה סימטרית
        SecretKeySpec secretKey = new SecretKeySpec(key.getBytes(), ALGORITHM);
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        cipher.init(Cipher.ENCRYPT_MODE, secretKey);
        return Base64.encodeToString(cipher.doFinal(data.getBytes()), Base64.DEFAULT);
    }
}

🏭 VMFactory – למה צריך Factory?

Android לא מאפשר ViewModel עם constructor parameters. VMFactory פותר את זה:

public class VMFactory implements ViewModelProvider.Factory {
    private final Context context;

    @Override
    public <T extends ViewModel> T create(Class<T> modelClass) {
        if (modelClass.isAssignableFrom(GameViewModel.class))
            return (T) new GameViewModel();
        if (modelClass.isAssignableFrom(StatisticsViewModel.class))
            return (T) new StatisticsViewModel(context);
        if (modelClass.isAssignableFrom(AchievementsViewModel.class))
            return (T) new AchievementsViewModel(context);
        throw new IllegalArgumentException("Unknown ViewModel");
    }
}

2️⃣ GameEngine – floodReveal (BFS/Recursion)▼
זהו BusinessLogic — לוגיקת המשחק. GameEngine לא יורש מ-Android כלל.
// חשיפת שטחים ריקים – BFS (Queue)
private void floodReveal(int startX, int startY) {
    Queue<int[]> queue = new LinkedList<>();
    queue.add(new int[]{startX, startY});

    while (!queue.isEmpty()) {
        int[] pos = queue.poll();
        int x = pos[0], y = pos[1];

        // בדיקת 8 שכנים
        for (int dy = -1; dy <= 1; dy++) {
            for (int dx = -1; dx <= 1; dx++) {
                if (dx == 0 && dy == 0) continue;
                int nx = x + dx, ny = y + dy;
                if (!isValidCoordinate(nx, ny)) continue;
                Cell neighbor = board[ny][nx];
                if (!neighbor.isRevealed() && !neighbor.isFlagged() && !neighbor.isMine()) {
                    neighbor.setState(CellState.REVEALED);
                    game.incrementTilesRevealed();
                    if (neighbor.getAdjacentMines() == 0)
                        queue.add(new int[]{nx, ny});
                }
            }
        }
    }
}
מבנה נתונים: Queue (תור) לאלגוריתם BFS. זה מבנה נתונים שהמורה ציינה ב-BusinessLogic (סעיף 9). BFS מבטיח שלא נקבל StackOverflow כמו ב-recursion על לוחות גדולים (30x16).
3️⃣ GameViewModel – finishGame + LiveData▼
public class GameViewModel extends ViewModel {
    private final GameRepository repository;
    private final MutableLiveData<GameStatus> gameState = new MutableLiveData<>();
    private final MutableLiveData<Game> currentGame = new MutableLiveData<>();
    private final MutableLiveData<Integer> timer = new MutableLiveData<>(0);

    public void finishGame(boolean isWin, long timeSeconds, String difficulty) {
        GameResult result = new GameResult(difficulty, timeSeconds, isWin);
        // 1. עדכון UI מיידי
        gameState.setValue(isWin ? GameStatus.WON : GameStatus.LOST);
        // 2. שמירה א-סינכרונית (לא חוסמת UI)
        repository.saveGameResult(result);
        // 3. חישוב XP
        XpManager.calculateAndAddXp(isWin, difficulty);
    }
}
תהליך: לחיצה על מוקש → GameEngine מחזיר LOST → ViewModel מעדכן LiveData → Activity רואה ומציג "Game Over" — הכל אוטומטי בלי לשאול "מה מצב המשחק?".

6️⃣ WorkManager – FirebaseSyncWorker▼
זהו הליך א-סינכרוני – הרחבה 1 שנדרשת. WorkManager > AsyncTask (deprecated).
public class FirebaseSyncWorker extends Worker {
    @Override
    public Result doWork() {
        // רץ ב-background thread, לא ב-UI thread
        List<GameEntity> unsynced = gameDao.getUnsynced();
        for (GameEntity g : unsynced) {
            firebaseManager.uploadGame(g);
            g.syncedToFirebase = true;
            gameDao.updateGame(g);
        }
        return Result.success();
    }
}

8️⃣ StatisticsViewModel – חישוב סטטיסטיקות▼
public class StatisticsViewModel extends ViewModel {
    public void loadStatistics() {
        isLoading.setValue(true);
        // Background thread — לא לחסום UI
        new Thread(() -> {
            List<Game> allGames = gameRepository.getAllGames();
            StatisticsData stats = calculateStatistics(allGames);
            statisticsData.postValue(stats); // postValue מ-background thread!
        }).start();
    }

    // חישוב Win Rate
    int gamesWon = 0, bestStreak = 0, tempStreak = 0;
    for (Game game : allGames) {
        if (game.getStatus() == GameStatus.WON) {
            gamesWon++; tempStreak++;
            if (tempStreak > bestStreak) bestStreak = tempStreak;
        } else { tempStreak = 0; }
    }
}
חשוב: setValue() רק מ-main thread. postValue() מ-background thread. טעות נפוצה בבגרות!

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read: if true;   // כולם קוראים
      allow write: if request.auth.uid == userId; // רק הבעלים כותב
    }
    match /scores/{scoreId} {
      allow read: if true;   // ציונים גלויים לכל
      allow create: if request.auth != null; // רק מחוברים מוסיפים
    }
  }
}
